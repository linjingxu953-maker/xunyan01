# 状态事件与确认接口设计

更新日期：2026-06-12

## 1. 设计目标

1. 定义完整的任务状态事件结构
2. 定义权限确认和记忆确认的接口
3. 解耦 UI、Agent、Core 三层
4. 为 Codex 提供明确的 UI 对接规范

### 2.2 事件模型

```csharp
public record TaskEvent
{
    public string EventId { get; init; } = Guid.NewGuid().ToString();
    public string TaskId { get; init; } = string.Empty;
    public TaskEventType EventType { get; init; }
    public string Message { get; init; } = string.Empty;
    public int Progress { get; init; }
    public DateTime Timestamp { get; init; } = DateTime.UtcNow;
    public Dictionary<string, object>? Metadata { get; init; }
}

public record ToolCallEvent : TaskEvent
{
    public string ToolName { get; init; } = string.Empty;
    public string ToolInput { get; init; } = string.Empty;
    public string? ToolOutput { get; init; }
    public bool IsStreaming { get; init; }
}

public record PermissionEvent : TaskEvent
{
    public PermissionType PermissionType { get; init; }
    public string PermissionScope { get; init; } = string.Empty;
    public string RequestReason { get; init; } = string.Empty;
    public PermissionDecision? Decision { get; init; }
}

public record MemoryEvent : TaskEvent
{
    public string MemoryType { get; init; } = string.Empty;
    public string MemoryContent { get; init; } = string.Empty;
    public MemoryDecision? Decision { get; init; }
}
```

### 2.3 事件流接口

```csharp
public interface ITaskEventStream
{
    /// <summary>
    /// 发布任务事件
    /// </summary>
    void Publish(TaskEvent evt);
    
    /// <summary>
    /// 订阅任务事件
    /// </summary>
    IObservable<TaskEvent> Subscribe(string taskId);
    
    /// <summary>
    /// 订阅所有事件
    /// </summary>
    IObservable<TaskEvent> SubscribeAll();
}
```

### 3.2 权限决策枚举

```csharp
public enum PermissionDecision
{
    AllowOnce,      // 允许本次
    AllowAlways,    // 永久允许（仅文件读取等低风险操作）
    Deny,           // 拒绝
    DenyAndRemember // 拒绝并记住
}
```

### 3.3 权限请求模型

```csharp
public record PermissionRequest
{
    public string RequestId { get; init; } = Guid.NewGuid().ToString();
    public string TaskId { get; init; } = string.Empty;
    public PermissionType PermissionType { get; init; }
    public string Scope { get; init; } = string.Empty;
    public string Reason { get; init; } = string.Empty;
    public string CommandOrPath { get; init; } = string.Empty;
    public RiskLevel RiskLevel { get; init; }
    public DateTime RequestedAt { get; init; } = DateTime.UtcNow;
}

public enum RiskLevel
{
    Low,      // 低风险：读取文件、读取窗口标题
    Medium,   // 中风险：写入文件、执行命令
    High      // 高风险：删除文件、执行危险命令
}
```

### 3.4 权限响应模型

```csharp
public record PermissionResponse
{
    public string RequestId { get; init; } = string.Empty;
    public PermissionDecision Decision { get; init; }
    public string? DenyReason { get; init; }
    public DateTime RespondedAt { get; init; } = DateTime.UtcNow;
}
```

### 3.5 权限确认接口

```csharp
public interface IPermissionPrompt
{
    /// <summary>
    /// 请求权限确认
    /// </summary>
    /// <param name="request">权限请求</param>
    /// <param name="ct">取消令牌</param>
    /// <returns>权限响应</returns>
    Task<PermissionResponse> PromptAsync(
        PermissionRequest request, 
        CancellationToken ct = default);
    
    /// <summary>
    /// 检查是否已有权限
    /// </summary>
    bool HasPermission(PermissionType type, string scope);
    
    /// <summary>
    /// 撤销权限
    /// </summary>
    void RevokePermission(PermissionType type, string scope);
}
```

### 3.6 UI 实现要点

UI 需要实现：

## 5. 依赖方向

```text
App (启动、DI)
  ↓
UI (权限弹窗、记忆弹窗、状态显示)
  ↓
Core (接口定义、模型)
  ↓
Agent (实现接口、调用确认)
```

关键原则：

1. Core 定义接口，不依赖 UI
2. UI 实现接口，通过 DI 注入
3. Agent 调用接口，不直接操作 UI
