# Expanded Production Prompt

## Title and Style

DesktopMascot first Douyin promo. Use the project design system from `design.md`: background `#100F14`, panels `#191820` and `#25232D`, text `#F7EBDD`, secondary `#BCAFA0`, amber `#FFB86B`, red `#D94848`, blue `#5FA8FF`, green `#73D39A`. Typeface strategy is renderer-safe `sans-serif` for Chinese headlines and `monospace` for interface/code texture.

## Rhythm

hook-PUNCH-build-PEAK-breathe-CTA across 28 seconds.

## Global Rules

9:16 vertical canvas, 1080 x 1920. It should look like a real desktop assistant was built in public over eight days, not like a generic SaaS slide deck. Use character artwork as foreground personalities, code/workflow panels as midground proof, and desktop grid/readout texture as background. Primary transition: blur/zoom through. Accent transition: overexposure flash into the product reveal. Every scene has entrance animations, ambient background motion, and no exit animations before transitions.

## Scene 1: Eight-Day Hook, 0.0-4.0s

Concept: The video opens like a confession from a tired builder. Huge text slams into a dark desktop stage while two assistant characters appear like recruited teammates.

Mood direction: late-night build log, cheeky Douyin hook, cinematic title sequence.

Depth layers: BG giant ghost `8 DAYS`, radial amber glow, thin desktop grid. MG massive Chinese hook text. FG two cutout characters, small labels for Codex/MiMo/Claude orbiting them, red punch marker on `牛马`.

Animation choreography: `8天` stamps in; hook text punches upward; characters rise from below as if summoned; labels snap to their sides; ghost text drifts slowly.

Transition out: blur zoom through, 0.35s, `power3.inOut`.

## Scene 2: Three Agents Building, 4.0-8.5s

Concept: Codex, MiMo Code, and Claude Code become a production line. Code strips, task chips, and commit counters cascade into a stack.

Mood direction: rapid desk montage, terminal energy, precise but not sterile.

Depth layers: BG terminal columns and commit ticks. MG three vertical agent lanes. FG day counter, code snippets, `8 DAYS / 3 AGENTS / 1 DESKTOP ASSISTANT`.

Animation choreography: agent names slide in from three directions; code lanes type on; counters count up; the central product name locks into place.

Transition out: directional blur upward, 0.32s, `power2.inOut`.

## Scene 3: Interactive Desktop Assistant, 8.5-13.5s

Concept: The build turns into an actual product. A floating assistant panel docks beside a screen-understand selection box, with a character watching the workflow.

Mood direction: practical product demo, UI glass, screen-aware assistant.

Depth layers: BG desktop window silhouettes and selection rectangle. MG floating chat/task card. FG character, action chips, pointer path.

Animation choreography: window silhouettes draw in; selection box scans; assistant card slides from right; chips cascade; character tilts forward.

Transition out: overexposure flash, 0.25s, `power4.inOut`.

## Scene 4: Workflow Spine, 13.5-19.0s

Concept: The assistant is not just a chat window; it plugs into workflows. A five-node pipeline runs from screen observation to memory and visible feedback.

Mood direction: workflow diagram as kinetic machinery, useful and controlled.

Depth layers: BG orbit rings and subtle node grid. MG pipeline nodes: screen, intent, tools, memory, character. FG permission badge, green completed pulse, moving execution token.

Animation choreography: nodes pop on in priority order; connector lines draw; execution token races through the path; permission badge flips from waiting to approved.

Transition out: zoom through, 0.35s, `expo.inOut`.

## Scene 5: Task Universe, 19.0-24.0s

Concept: Show the range without drowning viewers in text. Six large task tiles orbit the assistant: homework, online course, files, research, code, design/video.

Mood direction: fast Douyin feature burst, confident and playful.

Depth layers: BG ghost words and color pulses. MG orbiting task tiles. FG two characters, one working and one planning, as the task universe rotates around them.

Animation choreography: tiles burst from center; red/amber/blue accents sweep across; character cutouts float with parallax; task labels snap into readable positions.

Transition out: blur crossfade, 0.55s, `sine.inOut`.

## Scene 6: CTA and Product Name, 24.0-28.0s

Concept: The speed resolves into a clean identity frame. DesktopMascot is positioned as the first step toward a personal assistant that lives on the desktop.

Mood direction: calmer ending, creator roadmap, first-demo confidence.

Depth layers: BG soft amber and blue glows. MG product wordmark, subtitle, tiny build metadata. FG characters standing as a team and a final progress bar.

Animation choreography: wordmark draws in; subtitle floats upward; progress bar fills to `FIRST PROMO`; characters settle; final fade to black in the last half-second.

Transition out: final fade only.

## Recurring Motifs

Use `8 DAYS`, agent lanes, desktop selection rectangles, execution tokens, and character state art. Keep amber as the human/build accent, blue as system intelligence, red as hook emphasis, and green as completed workflow.

## Negative Prompt

Avoid generic AI sparkle, abstract robots, blue-purple neon dominance, tiny unreadable UI, centered static slides, and long explanatory paragraphs.
