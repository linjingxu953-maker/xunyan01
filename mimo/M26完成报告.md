## 1. 交付内容

### 1.1 TaskEventType 枚举

定义了 21 种事件类型：

```csharp
public enum TaskEventType
{
    // 任务生命周期
    TaskStarted,
    TaskCompleted,
    TaskFailed,
    TaskCancelled,

    // 步骤变化
    StepChanged,
    StepCompleted,

    // 工具调用
    ToolCallStarted,
    ToolCallCompleted,
    ToolCallFailed,

    // 权限请求
    PermissionRequested,
    PermissionGranted,
    PermissionDenied,

    // 记忆请求
    MemorySaveRequested,
    MemorySaveCompleted,
    MemorySaveRejected,

    // 进度更新
    ProgressUpdated,

    // 上下文读取
    ContextReadingStarted,
    ContextReadingCompleted,

    // LLM 调用
    LlmCallStarted,
    LlmCallCompleted,
    LlmStreamChunk
}
```

### 1.2 TaskEvent 模型增强

添加了以下功能：

1. `EventType` 属性 - 事件类型
2. `Metadata` 属性 - 元数据字典
3. `Timestamp` 属性 - 时间戳
4. 工厂方法：
   - `TaskEvent.TaskStarted()`
   - `TaskEvent.TaskCompleted()`
   - `TaskEvent.TaskFailed()`
   - `TaskEvent.ToolCallStarted()`
   - `TaskEvent.ToolCallCompleted()`
   - `TaskEvent.PermissionRequested()`
   - `TaskEvent.MemorySaveRequested()`
   - `TaskEvent.ProgressUpdated()`

### 1.3 ITaskEventStream 接口

```csharp
public interface ITaskEventStream
{
    void Publish(TaskEvent evt);
    IObservable<TaskEvent> Subscribe(string taskId);
    IObservable<TaskEvent> SubscribeAll();
    IReadOnlyList<TaskEvent> GetRecentEvents(string taskId, int count = 50);
}
```

### 1.4 TaskEventStream 实现

功能：
1. 基于 `Subject<T>` 的响应式事件流
2. 按任务 ID 分组存储事件
3. 自动清理旧事件（保留最近 500 条）
4. 支持按任务订阅和全局订阅

## 5. UI 对接说明

Codex 可以使用以下方式订阅事件：

```csharp
// 订阅指定任务的事件
_eventStream.Subscribe(taskId)
    .Subscribe(evt => {
        // 更新 UI 状态
    });

// 订阅所有事件
_eventStream.SubscribeAll()
    .Subscribe(evt => {
        // 全局事件处理
    });
```
