本计划的目标是先搭建一个可编译、可启动、可验证的 Windows 桌面悬浮小人 MVP，再逐步接入 Agent、上下文读取、权限和记忆能力。

## 1. MVP 定义

### 1.1 初版必须完成

第一阶段只做“可运行外壳 + 最小任务闭环”：

1. `.NET 8 + Avalonia 11.x` 桌面应用。
2. Windows 透明、无边框、置顶悬浮窗口。
3. 小人可拖动、可隐藏到托盘、可通过快捷键唤起。
4. 输入框可发送消息。
5. `TaskRouter -> AgentEngineStub -> TaskResult` 可完成一次本地假响应。
6. 状态机可驱动小人状态：`Idle / Listening / Working / Completed / Error`。
7. 所有代码可 `dotnet build`、`dotnet test`。

### 1.2 初版不做

这些能力先保留接口，不进入第一阶段实现：

1. 真实 OpenAI / Hermes Agent 调用。
2. OCR 屏幕读取。
3. 浏览器正文自动读取。
4. 文件写入和命令执行。
5. 长期记忆中心完整 UI。
6. Live2D / Lottie 完整动画系统。
7. 自动更新、插件市场、多角色。

### 1.3 第一阶段验收口径

第一阶段不能再声称“完整桌面 AI 小人”。正确口径是：

> 可运行的 Windows 桌面悬浮小人壳子，已打通 UI 输入、任务路由、假 Agent 响应和状态机。

## 2. 技术决策

| 项 | 决策 |
|---|---|
| 桌面框架 | C# / Avalonia UI 11.x |
| 运行时 | .NET 8 |
| UI 模式 | MVVM |
| MVVM 工具 | CommunityToolkit.Mvvm |
| 日志 | Serilog |
| 本地数据库 | SQLite，第二阶段接入 |
| 测试框架 | xUnit |
| 平台 | Windows 10 / Windows 11 优先 |

关键原则：

1. 先让工程稳定编译，再扩展功能。
2. 所有跨层通信通过接口，不让 UI 依赖具体服务类。
3. 高危能力先设计策略，后实现功能。
4. 示例代码必须能编译；无法编译的内容只写成设计说明。

## 6. Phase 2: 核心模型与接口

### 6.1 枚举

必须先定义：

1. `MascotState`
2. `TaskType`
3. `TaskStatus`
4. `PermissionLevel`
5. `PermissionDecision`

第一阶段的 `MascotState` 只需要：

```text
Idle
Listening
Working
Completed
Error
```

后续再加入：

```text
Understanding
ReadingContext
Planning
WaitingApproval
MemoryConfirm
Reporting
```

### 6.2 模型

第一阶段模型：

| 模型 | 必需字段 |
|---|---|
| `AgentTask` | `Id`、`Title`、`Input`、`Type`、`Status`、`CreatedAt` |
| `TaskResult` | `TaskId`、`Success`、`Content`、`Error`、`CompletedAt` |
| `TaskEvent` | `Id`、`TaskId`、`State`、`Message`、`Progress`、`CreatedAt` |

第二阶段模型：

| 模型 | 用途 |
|---|---|
| `PermissionRequest` | 文件、命令、记忆等确认请求 |
| `ContextSnapshot` | 当前窗口、选中文本、浏览器、截图等上下文 |
| `MemoryEntry` | 长期记忆条目 |

### 6.3 接口

接口必须覆盖 UI 需要的事件流，避免 UI 依赖具体类。

```text
ITaskRouter
  Task<TaskResult> DispatchAsync(AgentTask task, CancellationToken ct)

ITaskEventBus
  event EventHandler<TaskEvent> TaskEventPublished
  void Publish(TaskEvent evt)

IAgentEngine
  Task<TaskResult> ExecuteAsync(AgentTask task, CancellationToken ct)

IPermissionManager
  Task<PermissionDecision> RequestApprovalAsync(PermissionRequest request, CancellationToken ct)

IAppSettingsStore
  Task<AppSettings> LoadAsync(CancellationToken ct)
  Task SaveAsync(AppSettings settings, CancellationToken ct)
```

注意：不要把 `OnStateChanged` 只放在 `TaskRouter` 具体类上。UI 应订阅 `ITaskEventBus`，或者 `ITaskRouter` 接口明确暴露事件。

## 7. Phase 3: 最小 Agent 与任务流

### 7.1 AgentEngineStub

第一阶段必须实现一个假 Agent：

```text
AgentEngineStub
  输入: AgentTask
  输出: TaskResult
  行为: 延迟 300-800ms 后返回固定回复
```

这样可以先验证 UI、DI、状态机和任务流，不依赖外部 API。

### 7.2 TaskRouter

第一阶段流程：

```text
DispatchAsync
  -> 发布 Listening/Working 事件
  -> 调用 IAgentEngine.ExecuteAsync
  -> 根据结果发布 Completed/Error
  -> 返回 TaskResult
```

不要在第一阶段保存完整交互到数据库。可以先把任务历史保存在内存中，等 SQLite 阶段再落盘。

### 7.3 必须修正的编译风险

实现时必须确保：

1. `AgentEngineStub` 或 `AgentEngine` 类真实存在。
2. DI 注册的具体类型和实际类名一致。
3. `FloatingWindowViewModel` 只有一个主构造函数，通过 DI 注入所需服务。
4. UI 绑定的命令必须全部存在；暂不实现的菜单项先不要绑定。
5. 接口里不存在的事件不能在 ViewModel 中直接调用。

```text

  -> App.Initialize
```

## 10. Phase 6: 权限与安全

这一阶段在真实文件/命令能力前完成。

### 10.1 权限策略

| 等级 | 行为 | 初版策略 |
|---|---|---|
| L0 | 普通聊天 | 允许 |
| L1 | 当前窗口标题 | 首次确认 |
| L2 | 屏幕/浏览器读取 | 首次确认，可撤销 |
| L3 | 授权目录读取 | 目录级授权 |
| L4 | 文件写入 | 每次确认 |
| L5 | 命令执行 | 每次确认 + 策略检查 |
| L6 | 删除、支付、发送、发布 | 禁止 |

### 10.2 命令执行安全

即使用户确认，也不能直接执行任意命令。必须增加：

1. 命令风险解析。
2. 工作目录限制。
3. 禁止命令黑名单，例如 `del`、`Remove-Item -Recurse`、`format`、`shutdown`。
4. 环境变量脱敏。
5. 输出日志脱敏。
6. 每次执行单独确认，不保存永久授权。

第一阶段不实现命令执行，只实现策略接口和测试。

### 10.3 API Key 存储

不要把 API Key 保存在明文 JSON 或日志中。

Windows 初版建议：

1. 使用 Windows DPAPI 或 Credential Manager。
2. 设置页只显示掩码。
3. 日志过滤 `api_key`、`Authorization`、`Bearer`。
4. 导出日志默认不包含密钥和完整上下文正文。

## 11. Phase 7: 本地存储

### 11.1 存储范围

第一阶段不依赖数据库。

第二阶段加入：

1. SQLite 任务表。
2. SQLite 事件表。
3. SQLite 权限审计表。
4. JSON 设置文件。
5. Markdown/JSON 记忆文件。

### 11.2 数据库原则

1. 使用迁移脚本或版本号，不在构造函数里隐式改表。
2. `DateTime` 统一 UTC ISO 8601。
3. 任务上下文只存摘要，不默认存完整屏幕/网页正文。
4. 删除记忆时要同步删除索引和派生缓存。

## 12. Phase 8: 上下文读取

### 12.1 MVP 上下文

优先顺序：

1. 当前窗口标题和进程名。
2. 用户选中文本。
3. 用户手动粘贴网页内容。

### 12.2 延后上下文

以下能力复杂度高，延后：

1. OCR 屏幕读取。
2. 浏览器正文自动读取。
3. 浏览器扩展。
4. IDE 深度集成。

### 12.3 上下文读取原则

1. 用户可见。
2. 最小读取。
3. 明确来源。
4. 可撤销授权。
5. 不默认保存全文。
